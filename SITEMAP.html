User Journey & Data Flow

Login → Authentication Check ├─ Success: Load User Session & Permissions └─ Failure: Show Error Message ↓ Disease Selection Page Get user context, select disease pathway ↓ Main Workflow (Sequential) Features → Design Parameters → Simulation → Results → Analytics ↓ Each Page Accesses: 9 Internal Tabs Home, Design, Protocol, Materials, Toxicity, Cost, Delivery, Optimize, Quiz, 3D ↓ Data Operations Read/Write to: nanobio_studio.db, trial_registry.db, users.db ↓ External Services AI Engine, ToxCast API, Data Downloader ↓ Output: Results, Reports, Visualizations

Database Architecture

Main Databases: ├── nanobio_studio.db: Core application data ├── trial_registry.db: Simulation trials & results ├── users.db: User credentials & profiles └── ml_module.db: ML model training records Data Persistence Layer: ├── persistence.py: Session & state management ├── models.py: ORM models └── design_persistence.py: Design workflow state

Tab Features

  • Home Tab: Dashboard overview, quick stats, recent trials
  • Design Tab: Main design interface, component configuration
  • Protocol Tab: Step-by-step protocol development
  • Materials Tab: Material selection, composition, properties
  • Toxicity Tab: Toxicity predictions, safety data, risk assessment
  • Cost Tab: Pricing, budget analysis, ROI calculations
  • Delivery Tab: Delivery methods, targeting strategies, patient routes
  • Optimize Tab: Pareto optimization, multi-parameter tuning
  • Quiz Tab: Interactive learning, knowledge verification
  • 3D View Tab: 3D molecular visualization, rendering, exploration

User Roles & Access Control

👤 Admin

  • ✅ Full access to all pages (0-17)
  • ✅ Data sources management
  • ✅ ML model training & management
  • ✅ Database administration
  • ✅ User management
  • ✅ Audit dashboard & compliance
  • ✅ External data source configuration

👨‍💼 Designer

  • ✅ Main workflow pages (0, 1, 3, 4, 5)
  • ✅ All internal tabs (Design, Protocol, Materials, etc.)
  • ✅ AI Co-Designer features
  • ✅ Tutorial & learning resources
  • ✅ Trial history & analytics
  • ❌ Admin pages (12-15)

👁️ Viewer

  • ✅ Read-only trial history
  • ✅ View analytics & results
  • ✅ Tutorial & documentation
  • ❌ Design capabilities
  • ❌ Simulation execution
  • ❌ Admin features

RBAC Implementation

Files: rbac.py, auth.py, streamlit_auth.py Authentication Flow: User Login → Credential Verification ↓ Role Assignment & Session Creation ↓ Permission Check on Page Access ↓ Render UI based on Role: ├─ Admin: Show all UI elements ├─ Designer: Show workflow + tabs └─ Viewer: Show read-only views
